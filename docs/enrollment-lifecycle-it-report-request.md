# Enrollment Lifecycle IT Report Request

## Purpose

This document describes a future IT/Argos report or export needed to support full Enrollment Lifecycle analysis in TIMBER Enrollment Analytics.

The requested report should capture enrollment at key term milestones:

- First Day of Class enrollment
- Census 1 enrollment
- Census 2 enrollment, when applicable
- Final enrollment

## Desired Fields

- Term
- CRN
- Subject
- Course
- Section
- Course title
- Division
- Department
- Campus
- Building
- Room
- Instructional method / modality
- Start date
- End date
- Days
- Start time
- End time
- Instructor
- Capacity
- Waitlist
- First Day Enrollment
- Census 1 Enrollment
- Census 2 Enrollment
- Final Enrollment
- FTES, if available
- Accounting method
- Weekly contact hours
- Total contact hours

## Why This Report Is Needed

The lifecycle report would allow enrollment analysis to distinguish enrollment loss before census from enrollment loss after census. Current census-to-final attrition analysis is useful, but it cannot show whether students are leaving before instruction begins, shortly after the first day, between census points, or after census.

This report would help the college better understand enrollment behavior after the first day of instruction and support scheduling, retention, late-start planning, and enrollment management decisions.

It would also help identify courses that lose students early versus courses that lose students after census. That distinction matters because the possible interventions are different. Early loss may point to registration, communication, section timing, or first-week attendance issues, while post-census loss may point to persistence, course difficulty, support needs, or other retention factors.

## Forward-Looking Note

This is a forward-looking report request. Historical milestone data may not exist or may not be available in a consistent export format.

The application should support these fields going forward once IT can provide them. Until then, lifecycle milestone values should remain optional and display as N/A when missing.
